## 1. 启动与配置

### 1.1 启动服务

**方法一：双击 `start_server.bat`（Windows）**
```bash
# 自动启动 HTTP 服务，打开浏览器
# 默认地址：http://127.0.0.1:8080
```

**方法二：命令行启动**
```bash
cd 项目目录
python start_server.py
```

**方法三：自定义端口**
```bash
python start_server.py --port 8080
python start_server.py --port 8080 --host 0.0.0.0  # 局域网访问
python start_server.py --file my_prompts.html       # 指定 HTML 文件
```

### 1.2 首次打开

启动后浏览器自动打开页面。如果端口被占用，命令行会提示输入新端口。

## 2. 提示词生成

### 2.1 界面布局

页面从上到下分为：

```
┌─ 提示词输入区 ─────────────────────┐
│  🇨🇳 中文  [输入/显示提示词]         │
│  🇬🇧 English [自动同步显示]          │
│  [🎲 随机] [✨ 优化] [🤖 AI] [📋 复制] [清空] │
└──────────────────────────────────────┘
┌─ AI 润色控制 ──────────────────────┐
└─ ComfyUI 控制 ─────────────────────┐
┌─ 身份栏 ───────────────────────────┐
│  年龄 [====o====] 25 [🧪 随机]     │
│  地区 [中国 ▼]         [🧪 随机]   │
│  性别 [女 ▼]           [🧪 随机]   │
└──────────────────────────────────────┘
┌─ 提示词类别卡片 ───────────────────┐
│  [场景] [风格] [光线] [色彩] [画面] │
└──────────────────────────────────────┘
┌─ 图片画廊 ─────────────────────────┐
└─ 数据导出/导入 ────────────────────┘
```

### 2.2 随机生成

1. 在类别卡片中**开启**需要参与的类别（开关滑块）
2. 点击 **🎲 随机** 按钮
3. 系统从已开启类别中各随机取一个词条，组合成提示词
4. 中文框显示带有身份前缀的提示词，英文框自动同步

**示例输出：**
```
中文：中国，25岁，女，山水，写实，黄金时刻，浅景深
English：Chinese, 25 years old, Female, landscape, realistic, golden hour, shallow depth of field
```

### 2.3 模板优化

点击 **✨ 优化**，系统自动在提示词前添加：
- 随机选择一条**润色模板**（如"电影质感胶片摄影…"）
- 随机选择一条**皮肤描述**（如"皮肤细腻真实…"）
- 随机选择一条**画质描述**（如"8K 超高清…"）

### 2.4 AI 润色

需先配置 AI 厂商（见第 6 章）。开启 **🤖 AI 润色** 开关后，点击 **🤖 AI** 按钮：
1. 系统会带上预设（系统提示词）发送给 AI
2. AI 返回优化后的中英文提示词
3. 自动填入中文框和英文框
4. 记录保存到 AI 润色历史

### 2.5 复制与清空

- **📋 复制**：分别复制中文或英文框内容到剪贴板
- **清空**：清空两个文本框
- **快捷键**：`Ctrl + Enter` 快速随机生成

## 6. AI 润色

### 6.1 配置 AI 厂商

点击 AI 控制栏的 **⚙️ 设置**：

```
┌─ AI 设置 ──────────────────────────┐
│ AI 厂商 [DeepSeek ▼]                │
│ API 地址 [https://api.deepseek.com] │
│ API Key [••••••••••••••••]         │
│ 模型名称 [deepseek-chat] [📡 获取模型] │
│ [🔌 测试连接]                       │
│ [💾 保存设置]                       │
└──────────────────────────────────────┘
```

**预设厂商：**

| 厂商 | 自动填充地址 | API Key |
|------|------------|---------|
| DeepSeek | `https://api.deepseek.com` | 必填 |
| Kimi (月之暗面) | `https://api.moonshot.cn/v1` | 必填 |
| LLaMA | `https://api.llama-api.com` | 必填 |
| OpenAI | `https://api.openai.com/v1` | 必填 |
| Ollama (本地) | `http://localhost:11434/v1` | 可选 |
| LM Studio (本地) | `http://localhost:1234/v1` | 可选 |
| 自定义 | 用户填写 | 按需 |

**操作步骤：**
1. 选择厂商 → API 地址自动填充
2. 填写 API Key（本地模型可留空）
3. 填写模型名称，或点击「📡 获取模型」自动拉取列表
4. 点击「🔌 测试连接」验证配置
5. 点击「💾 保存设置」

### 6.2 设置润色预设

点击 **📝 预设**，编辑系统提示词（system prompt）：

```
💡 该预设只在首次通信或通信中断时发送一次

[编辑框：系统提示词内容]

[💾 保存预设]
```

默认预设会告诉 AI 以指定格式返回中英文润色结果。

### 6.3 使用 AI 润色

1. 在 AI 控制栏开启 **🤖 AI 润色** 开关
2. 输入或随机生成提示词
3. 点击 **🤖 AI** 按钮
4. 等待 AI 返回结果（按钮显示「⏳ AI 润色中…」）
5. 润色结果自动填入中文框和英文框

### 6.4 查看润色历史

点击 **📜 历史**，查看最近 20 条 AI 润色记录（原始/润色后中英文对照）。

## 7. ComfyUI 生图

### 7.1 连接检测

1. 启动 ComfyUI（默认端口 8188）
2. 点击 **🔌 检测连接** 自动检测
3. 状态显示 🟢 已连接 / 🔴 未连接

> **跨域问题**：如果通过 HTTP 服务打开页面，开启 **⚙️ 设置** → **「使用代理」** 开关，所有请求通过同源代理转发，解决 CORS 问题。

### 7.2 工作流管理

**准备工作流：**
1. 在 ComfyUI 中搭建好工作流
2. 点击 ComfyUI 的「Save」或「Export」→ 导出为 JSON 文件
3. 在本页面点击 **📂 上传工作流** 选择该 JSON 文件

**工作流格式支持：**
- **保存格式**：`{nodes: [...], links: [...]}`（ComfyUI 界面直接 Save）
- **API 格式**：`{node_id: {class_type: "...", inputs: {...}}}`（Save API Format）

**系统会自动：**
- 识别 CLIPTextEncode 节点（用于注入提示词）
- 识别 EmptyLatentImage 节点（用于读取默认尺寸）
- 识别 KSampler / KSamplerAdvanced 节点（用于随机种子）

### 7.3 生图设置

点击 **⚙️ 设置**：

```

│ [🧪 使用代理]                        │
│                                     │
│ 📂 工作流管理                        │
│ [📤 上传工作流 JSON]                 │
│ - my_workflow.json  [选择] [删除]   │
│                                     │
│ 🔍 已检测 2 个 CLIPTextEncode 节点  │

│                                     │
│ 🌐 发送语言 ○ English 英文 ● 中文 Chinese │

# 🖼️ ComfyUI 生图操作说明

## 准备工作流（重要！）

**工作流必须使用 API 格式导出：**

1. 在 ComfyUI 中搭建好工作流
2. 点击 ComfyUI 右上角 **Save (API Format)** 按钮 → 导出为 JSON 文件
3. ✅ **正确操作**：点击 Save (API Format)
4. ❌ **不要用普通的 Save/Export**（那种格式包含界面布局信息，本工具无法解析）

## 上传工作流

1. 启动 ComfyUI 后，在本页面点击 **🔌 检测连接**
2. 状态显示 🟢 已连接
3. 点击 **📂 上传工作流** → 选择刚刚导出的 API 格式 JSON 文件
4. 系统会自动识别：
   - CLIPTextEncode 节点（用于注入提示词）
   - EmptyLatentImage 节点（读取图片尺寸）
   - KSampler / KSamplerAdvanced 节点（控制随机种子）

## 设置与生图

1. 点击 **⚙️ 设置** → 设置宽高、语言、批次数量
2. 开启需要参与的**提示词类别**（勾选开关）
3. 点击 **🚀 开始生图**

进度显示：`🎲 种子 1234567 → 🎨 生成中... 5/9步 → ✅ 已完成`

**参数说明：**

| 参数 | 说明 |
|------|------|
| **发送语言** | 选择发送到 ComfyUI 的提示词语言 |
| **随机提示词数量** | >1 时自动从已开启类别生成多组不同提示词 |
| **生图批次** | 每组提示词生成 N 张（不同随机种子） |
| **尺寸** | 覆盖工作流中的 EmptyLatentImage 宽高 |

### 7.4 开始生图

点击 **🚀 开始生图**：

```
完整流程：
第 1 组 → 随机生成提示词A → ✨ 优化 → 🤖 AI 润色（如开启） 
       → 🎲 种子 1234567 → 发送 ComfyUI → 生成 → 显示图片
       → 🎲 种子 7654321 → 发送 ComfyUI → 生成 → 显示图片
       → 🎲 种子 3456789 → 发送 ComfyUI → 生成 → 显示图片
       → 🎲 种子 9876543 → 发送 ComfyUI → 生成 → 显示图片
第 2 组 → 随机生成提示词B → ...（重复）

进度显示：
🎲 种子 18273645 (1/8)
🎨 生成中... 5/9步
✅ 已完成 4/8
```

### 7.5 中止生图

生成过程中 **⏹ 中止** 按钮会显示，点击后：
- 当前正在生成的批次会继续完成
- 后续未开始的批次取消
- 按钮变为「🔄 重新开始」

## 9. 数据管理

### 9.1 自动保存机制

所有数据在修改时自动保存：

```
用户操作 → localStorage（浏览器） → syncAllToServer → data/*.json（服务器）
```

服务器保存的文件：

| 文件 | 保存内容 |
|------|---------|
| `data/prompts.json` | 提示词、类别、身份设置、润色模板 |
| `data/ai_settings.json` | AI 厂商、Key、模型、预设、历史 |
| `data/comfy_settings.json` | ComfyUI 地址、工作流、画廊 |
| `data/server_config.json` | 端口、主机地址 |

### 9.2 导出数据

点击底部的 **提示词生成器导出**：

```
选择要导出的文件（输入编号）：

1. 提示词数据（prompts）
2. AI 润色设置（ai_settings）
3. ComfyUI 设置（comfy_settings）
4. 服务配置（server_config）
```

导出文件名自动包含类别和时间戳。

### 9.3 导入数据

点击 **提示词生成器导入**，选择之前导出的 JSON 文件。系统会根据文件名自动识别类别：

| 文件名包含 | 自动导入 |
|-----------|---------|
| `prompts_` | 提示词数据 |
| `ai_` | AI 润色设置 |
| `comfy_` | ComfyUI 设置 |
| `server_config_` | 服务配置 |

如果文件名无法识别或文件包含多种数据，会弹出选择框让你选。

### 9.4 页面刷新后数据恢复

```
打开页面 → init()
  ↓
loadAllFromServer() → 从服务器 JSON 文件加载
  ├─ 成功 → 写入 localStorage → 渲染
  └─ 失败 → 从 localStorage 加载 → 渲染
  ↓
loadDataConstants() → 加载 cn_en_map / identity_en / enrich_data
```

## 10. 常见问题

### 10.1 ComfyUI 连接不上

```
检查项：
□ ComfyUI 是否已启动？
□ 端口是否正确（默认 8188）？
□ 是否开启了「使用代理」？
□ 是否通过 HTTP 服务打开页面（非 file://）？
```

### 10.2 AI 润色失败

```
可能原因：
□ API Key 是否正确？
□ 模型名称是否正确？
□ 网络是否能访问 API 地址？
□ 点击「🔌 测试连接」验证
```

### 10.3 数据丢失

```
恢复步骤：
1. 如果有备份文件 → 点击「提示词生成器导入」
2. 如果之前通过服务器运行过 → data/ 目录下应该有备份 JSON
3. 如果都没有 → 重新配置，建议先导出备份
```

### 10.4 端口被占用

启动时如果端口被占用，命令行会提示：
```
⚠️ 端口 8080 已被占用
   请输入新端口 (直接回车使用 8081):
```
输入新端口或直接回车使用 +1 端口。

### 10.5 页面无样式/空白

```
可能原因：
□ 是否通过 file:// 直接打开 HTML？
   → 必须通过 python start_server.py 启动 HTTP 服务
□ 服务器端口是否和浏览器访问的一致？
   → 检查命令行输出的端口号
```

### 10.5 无法进行生图
```
可能原因：
□ 上传工作流后提示"未找到 CLIPTextEncode 节点
  →工作流是用普通 Save 导出的，不是 Save (API Format)。请重新用 API 格式导出。

□ 所有图片种子相同
  → KSampler 的 seed 必须是 Widget（不要连接到其他节点），系统会自动每批次生成不同随机种子。
